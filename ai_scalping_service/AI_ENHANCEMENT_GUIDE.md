# AI-Enhanced Index Scalping Service v5.0

## 🚀 Overview

This service has been enhanced with world-class AI-powered trading capabilities targeting **90%+ win rate**. The system integrates Google Gemini AI for intelligent trade validation, multi-instrument support, and institutional-grade capital management.

---

## 🤖 Key Features

### 1. Gemini AI Prediction Engine
- **Location**: `core/gemini_prediction_engine.py`
- **Purpose**: Leverages Google Gemini 2.0 for high-confidence trade predictions
- **Target Win Rate**: 90%+

**Features**:
- Multi-layer validation (5 layers must align)
- Real-time market data analysis
- Confidence scoring with breakdown
- Price target and stop loss predictions
- Veto system for low-confidence conditions

### 2. Instrument Selection System
- **Location**: `core/instrument_selector.py`
- **Purpose**: Dynamic selection of trading instruments

**Supported Instruments**:
| Instrument | Exchange | Lot Size | Freeze Qty | Expiry Day |
|------------|----------|----------|------------|------------|
| NIFTY      | NSE      | 75       | 1,800      | Thursday   |
| BANKNIFTY  | NSE      | 35       | 1,050      | Wednesday  |
| SENSEX     | BSE      | 20       | 1,000      | Friday     |
| BANKEX     | BSE      | 30       | 900        | Monday     |

**Allocation Strategies**:
- `equal` - Equal capital to all instruments
- `volatility_weighted` - More capital to stable instruments (default)
- `inverse_volatility` - More capital to volatile instruments
- `margin_weighted` - Based on margin requirements
- `custom` - User-defined weights

### 3. AI Orchestrator
- **Location**: `core/ai_orchestrator.py`
- **Purpose**: Coordinates AI predictions with trading engines

**Trade Validation Flow**:
1. Signal generated by scalping engine
2. AI validates with Gemini prediction
3. If confidence >= 85%, trade proceeds
4. If confidence < 85% or veto triggered, trade skipped
5. Position sizing adjusted based on confidence

### 4. Institutional Capital Manager
- **Location**: `core/capital_manager.py`
- **Purpose**: Institutional-grade position sizing

**Features**:
- Kelly Criterion-based sizing
- VIX-adjusted position scaling
- Order slicing for large positions
- Daily loss limit enforcement
- Win rate-based allocation

## 📡 API Endpoints

### Instrument Selection

```
GET  /instruments              - List all available instruments
GET  /instruments/active       - Get currently active instruments
POST /instruments/select       - Select which instruments to trade
POST /instruments/toggle/{inst} - Toggle specific instrument on/off
```

### AI Prediction

```
GET  /ai/status                - Get AI orchestrator status
POST /ai/configure             - Configure AI settings
POST /ai/validate_trade        - Validate a trade with AI
GET  /ai/prediction/{inst}     - Get AI prediction for instrument
GET  /ai/stats                 - Get AI validation statistics
POST /ai/reset_stats           - Reset AI statistics
```

### Capital Management

```
GET  /capital                  - Get capital status
POST /capital/configure        - Configure capital settings
POST /capital/update_premiums  - Update live ATM premiums
GET  /capital/position_size/{inst} - Calculate position size
POST /capital/allocate         - Allocate capital
POST /capital/record_trade     - Record trade result
POST /capital/reset_daily      - Reset daily stats
```

### Combined Configuration

```
POST /trading/configure        - Configure all settings at once
```

## 🎯 Usage Examples

### 1. Select Instruments to Trade

```bash
curl -X POST http://localhost:8003/instruments/select \
  -H "Content-Type: application/json" \
  -d '{
    "instruments": ["NIFTY", "BANKNIFTY"],
    "allocation_strategy": "volatility_weighted"
  }'
```

### 2. Configure AI Settings

```bash
curl -X POST http://localhost:8003/ai/configure \
  -H "Content-Type: application/json" \
  -d '{
    "ai_enabled": true,
    "min_confidence_threshold": 0.90,
    "prediction_mode": "conservative"
  }'
```

### 3. Get AI Prediction

```bash
curl http://localhost:8003/ai/prediction/NIFTY?spot_price=24500&vix=14.5
```

### 4. Validate Trade Before Execution

```bash
curl -X POST "http://localhost:8003/ai/validate_trade?\
instrument=NIFTY&\
signal_type=CE_BUY&\
entry_price=150&\
quantity=10&\
target_price=200&\
stop_loss=120&\
spot_price=24500&\
vix=14.5"
```

### 5. Configure All Settings at Once

```bash
curl -X POST http://localhost:8003/trading/configure \
  -H "Content-Type: application/json" \
  -d '{
    "capital": 1000000,
    "instruments": ["NIFTY", "BANKNIFTY", "SENSEX"],
    "allocation_strategy": "equal",
    "ai_enabled": true,
    "min_confidence": 0.85
  }'
```

## 🛡️ Risk Management

### Veto Conditions (NO_TRADE)
- Confidence below threshold
- VIX > 25 (too volatile)
- First 5 minutes after market open (9:15-9:20)
- Last 15 minutes before close (3:15-3:30)
- Major events pending
- Conflicting signals across layers
- Risk/reward ratio < 1.5

### Position Sizing Rules
- Maximum 25% of capital per position
- VIX adjustment: reduce size in high volatility
- Confidence scaling: smaller size for lower confidence
- Order slicing: split large orders to avoid market impact
